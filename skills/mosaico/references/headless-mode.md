# Headless Mode

Read this when Mosaico says headless mode is on or changes.

## Delivery Contract

Headless mode is about where ordinary text output goes. When it is on, use the
relevant channel to deliver anything another participant should receive. When
it is off, ordinary text output is visible in the session.

Headless mode does not change scope, authority, privacy, or the need to publish
information that belongs in shared coordination.

## Work Deliberately

- Begin working without an acknowledgement message or running narration.
- Publish a self-contained update when you need judgment, coordination, a
  decision, a blocker, a handoff, a meaningful milestone, failure reporting, or
  completion.
- Deliver a direct answer requested by the human in a channel before ending the
  turn. A local final response is not delivery while headless mode is on.
- Keep private reasoning, raw logs, and routine tool chatter out of channels.

## Route The Update

- Reply to the message that started the work when it remains the right context.
- Otherwise use the narrowest relevant channel. Read [Channel Creation](channel-creation.md)
  if the work needs a focused place to continue.
- Read [Coordination Guide](coordination-guide.md) before choosing a reply,
  send, reaction, tag, handoff, or participant.
- Treat successful channel publication as delivery. If it fails, retry or route
  the failure through an available channel; do not assume ordinary output was
  seen.

## Handle A Mode Change

- When headless mode turns on, publish any consequential context that has not
  already reached the relevant channel, then continue with normal publication
  cadence.
- When it turns off, continue publishing shared decisions and cross-agent
  coordination, but do not duplicate routine conversational output into chat.
