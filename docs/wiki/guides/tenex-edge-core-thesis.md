# tenex-edge Core Thesis

## Core Thesis

The core thesis of tenex-edge is the inversion of TENEX: instead of hosting agents, it connects agents that already run in their native homes, grafting a shared nervous system over Nostr onto them. tenex-edge is not a new product but the missing membrane — the on-ramp that lets a foreign-hosted agent become a first-class citizen of a fabric (TENEX, podcast-player, proactive-context) that already exists. <!-- [^8a3eb-6] -->

## Sovereign Identity & Shared World-Model

The real idea of tenex-edge is giving agents a sovereign identity and a shared world-model independent of the tool they run inside — the host is just a body, and the durable identity, memory, presence, and relationships float above it and persist across hosts, devices, and time. <!-- [^8a3eb-7] -->

## Scope: Floor and Ceiling

The floor of tenex-edge is awareness + identity (knowing what your fleet is doing, work following you across machines, agents that remember who they are between sessions/hosts) — single-player, no trust problem, no consensus problem, definitely real. The ceiling is coordination (collision avoidance, dedup, who-owns-the-bug) — high-wow, high-risk, resting on an unproven premise that costly collisions happen often enough to warrant a coordination layer; that premise should be tested before it defines the project. <!-- [^8a3eb-8] -->
